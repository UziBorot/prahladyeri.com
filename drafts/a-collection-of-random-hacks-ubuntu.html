Whether you are a sysadmin, developer or just a user, if you face a non-trivial issue while setting up any software, you generally head over to sites like stackoverflow.com to find any solutions. If the problem is solved and you are happy, you generally do two things: Bookmark/Star that post, and secondly make a note to your own workflow about that issue for future references. For what its worth, here is the collection of such notes prepared by me for ubuntu 12.04 LTS. However, most solutions should work on 14.04 too.

To boot ubuntu to command prompt intead of GUI

  1. Modify your file /etc/default/grub with GRUB_CMDLINE_LINUX_DEFAULT="text". Optionally, also uncomment this line:

    #GRUB_TERMINAL=console

    and do sudo update-grub. The machine will boot in text mode.

  2. To start the graphical interface just do

    sudo service lightdm start #could be gdm or startx in some situations

Create a private repo based on a previously burned aptoncd backup (.ISO file)

  1. Create a mount point in /etc/fstab that points to that ISO file on your file system:\ /path/to/iso /media/mount-point auto defaults 0 0
  2. Add this line in /etc/apt/sources.list: deb [arch=i386] file:/media/repo-name/ /
  3. Just perform a sudo apt-get update
  4. To upgrade packages, don’t use the ubuntu GUI, instead do sudo apt-get upgrade

Resetting a modem/usb-device using usb-modeswitch

We can try if a reset of our USB device helps resurrecting it. We first need to find out the vendor ID, and the product ID easiest by\ lsusb.

This will give information e.g. like follows:\ Bus 003 Device 002: ID 04f9:0224 Brother Industries, Ltd

In this example the vendor ID is 04f9, product ID is 0224 (right, its my printer).\ To reset the device we may then issue\ sudo usb_modeswitch -R -v -p

      enable service at startup

      sudo rm /etc/init/mysql.override```

  • list all services: initctl list

iptables basics:

  • Save/Restore state:\ iptables-save > /etc/iptables.rules #save current state to file\ iptables-restore \< /etc/iptables.rules #restore state from file
  • Save state (using iptables-persistent package):
    • method-1: install the package and run: sudo /etc/init.d/iptables-persistent save
    • method-2 for IPv4 rules: sudo su -c iptables-save > /etc/iptables/rules.v4
    • method-3 for IPv6 rules: sudo su -c ip6tables-save > /etc/iptables/rules.v6
  • Auto load on network up - Append this command to an iface config in /etc/network/interfaces:\ post-up iptables-restore \< /etc/iptables.rules
