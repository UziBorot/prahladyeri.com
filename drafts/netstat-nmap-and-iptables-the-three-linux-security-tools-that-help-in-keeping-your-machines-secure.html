netstat, nmap and iptables - The three wonderful Linux security tools that help in keeping your machines secure

Tweet
Published: Sun 08 October 2017
By Prahlad Yeri

In Linux.

tags: Linux Open Source

Today, I'd like to mention about three linux security tools that go a long way towards securing your linux devices. In fact, these three are easily the top linux tools when you want to audit the most basic aspects of linux security:

  1. Ensure there are no listening ports on your device (other than those that you want).
  2. Ensure there are no back-doors running on your device that are communicating with a remote device port.

netstat is easily the best tool for this job. For example, the command sudo netstat -antp on this Ubuntu machine shows that the only active tcp connections are those created by the firefox process:

netstat tool running on Ubuntu 16.04 LTS

Similarly, the sudo netstat -anup command (without grep filter) shows that the only udp connections open are those created by dnsmasq.

nmap tool, on the other hand, also does the same thing (port scanning), but can do it on a remote machine on your network too. You can use it to confirm the same facts as above from the perspective of another machine on your local area network. Besides that, you can use it to scan for open ports on your other device too. For example, you can use the nmap tool from your PC/laptop to test for any open listening ports on your android phone on the same WiFi network as yours:

nmap tool running on Ubuntu 16.04 LTS

Finally, iptables is the most important of all as it is the linux firewall that allows you to block in/out traffic on specific ports. For example, if you don't host any web apps or run any service such as ssh-server on your device (on most personal computers, you won't), then you can block all incoming traffic using the iptables command as thus:

/sbin/iptables -P INPUT DROP

However, you can allow specific incoming ports, for example the following if you allow incoming traffic on http, https and ssh:

/sbin/iptables -A OUTPUT -p tcp --match multiport --dports 80,443,22 -j ACCEPT #http, https, ssh

Its also advisable to disable all forwarding on your machine using iptables (though Ubuntu usually does this by default in the stock kernel settings too):

/sbin/iptables -P FORWARD DROP

The simplicity of iptables helps you not just in security your own personal computer, but any cloud instances you might have, for example, on Amazon AWS or Digital Ocean.
